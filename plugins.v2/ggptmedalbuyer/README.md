# GGPT 勋章购买

> **来源**：本插件移植自 [jiangbkvir/MoviePilot-Plugins](https://github.com/jiangbkvir/MoviePilot-Plugins) 的 `ggptmedalbuyer`（原作者 `jiangbkvir`），由 `wuyaos` 在此基础上修复到期时间解析缺陷。
>
> **v1.0.1 修复**：原版解析到期时间时误用勋章 ID 全页搜索，`find("35")` 会命中「分享率: 3.935」里的 35，导致取到错误勋章的结束日期而判定「该买没买」。本版改为按购买按钮 `data-id` 锚定单行解析、个人中心按 `title=勋章名` 的 `<img>` 精确匹配到期时间。

自动检查并续购 GGPT 站点的“疯狂星期四”勋章，并按有效期计算下一次购买时间。

## 功能

- 默认购买“疯狂星期四”勋章，勋章 ID 为 `35`
- 自动读取 MoviePilot 站点管理中的 GGPT 站点信息
- 每天 `08:00` 检查一次勋章页面，已购买时只更新预计下次购买时间
- 勋章页面显示已购买时，自动读取个人中心里的勋章过期时间
- 到达预计购买时间后再次检查，确认未购买或已到期才购买勋章
- 支持到期后延迟指定秒数再购买
- 插件配置页管理启用、提醒、勋章 ID 和偏移量
- 购买成功或失败后发送通知
- 插件详情页展示 UID、站点、到期时间、预计下次购买时间、勋章持有状态和购买记录卡片
- 购买记录同时展示成功和失败结果

## 使用前准备

请先在 MoviePilot 的站点管理中维护 GGPT 站点：

```text
站点名称：GGPT
站点域名：gamegamept.com
站点地址：https://www.gamegamept.com/
```

站点 Cookie 需要保持有效，否则插件无法读取勋章页面或提交购买请求。

## 配置说明

| 配置项 | 默认值 | 说明 |
| --- | --- | --- |
| 启用插件 | 关闭 | 开启自动刷新和到点购买 |
| 购买成功提醒 | 开启 | 购买成功或失败后发送通知 |
| 勋章 ID | `35` | “疯狂星期四”勋章当前 ID |
| 偏移量（秒） | `0` | 到期后延迟多少秒购买 |
| 预计下次购买时间 | 详情页展示 | 插件计算出的下一次购买时间 |
| GGPT 站点状态 | 详情页展示 | 显示 GGPT 站点读取状态 |
| UID | 详情页展示 | 从 Cookie 解析 |
| 是否有疯狂星期四勋章 / 到期时间 | 详情页展示 | 从勋章页或个人中心页面解析 |

配置项在 MoviePilot 原生插件配置弹窗中保存。保存后插件会自动检查一次：已购买时只刷新预计下次购买时间，未购买或已到期时才提交购买。

## 运行逻辑

插件会访问 GGPT 勋章页面：

```text
https://www.gamegamept.com/medal.php
```

从页面中解析勋章 ID、勋章名称、有效期和是否已经购买。当前“疯狂星期四”的有效期为 `7` 天。

以下入口都会执行同一套流程：先检查页面，再判断是否购买。

- 保存插件配置
- 每天 `08:00` 自动检查
- 到达预计下次购买时间的自动任务

如果页面显示该勋章已经购买，插件不会提交购买接口，只会计算并更新预计下次购买时间。

已购买时，插件会从站点 Cookie 的 `c_secure_pass` 中解析当前用户 ID，并访问个人中心页面读取勋章真实过期时间：

```text
https://www.gamegamept.com/userdetails.php?id=当前用户ID
```

用户 ID 不需要在插件里填写。

预计下次购买时间按以下规则计算：

1. 如果能从勋章页或个人中心解析到当前勋章到期时间，使用 `到期时间 + 偏移量秒数`。
2. 如果页面显示已经购买但解析不到精确到期时间，优先使用已保存的未来到期时间。
3. 如果有成功购买记录，使用 `最近购买时间 + 7 天 + 偏移量秒数`。
4. 如果页面显示已经购买但没有历史成功记录，使用 `当前检查时间 + 7 天 + 偏移量秒数` 作为兜底预计时间。
5. 如果页面未显示已购买，且没有可用到期时间，才会尝试购买一次。
6. 如果购买失败且无法计算真实到期时间，会显示下一次每日检查时间。

插件不会每分钟轮询。它会按预计购买时间注册到点购买定时器，到点后自动购买。

## 购买接口

确认需要购买时插件会提交：

```text
POST https://www.gamegamept.com/ajax.php

action=buyMedal&params[medal_id]=35
```

接口返回 `ret=0` 表示购买成功；`ret` 非 0 表示购买失败。

## 通知内容

通知示例：

```text
【GGPT勋章购买】
站点：GGPT
勋章：疯狂星期四
勋章 ID：35
状态：购买成功 / 购买失败
说明：接口返回说明
预计下次购买：YYYY-MM-DD HH:mm:ss
```

常见失败说明：

| 提示 | 含义 |
| --- | --- |
| G 值/魔力不足，无法购买勋章 | 账号余额不足 |
| 登录态无效，请更新 MoviePilot 站点管理中的 GGPT Cookie | Cookie 过期 |
| 权限不足或登录态无效，请检查 GGPT Cookie | 站点拒绝请求 |

## 购买记录

插件详情页以卡片展示当前状态和最近 30 条购买记录，桌面端按 6 个一行排列，成功和失败都会展示。没有记录时会显示“暂无购买记录”。

如果插件没有写入过购买接口记录，但已经从页面或个人中心读取到当前勋章到期时间，详情页会展示一张“已购买”状态卡片，避免手动购买或历史购买后页面空白。

| 字段 | 说明 |
| --- | --- |
| 时间 | 购买请求完成时间 |
| 站点 | GGPT |
| 勋章名称 | 购买的勋章名称 |
| 状态 | 购买成功、购买失败、Cookie 失效或配置错误 |
| 说明 | 接口返回或插件判断出的说明 |

## 常见问题

### 为什么购买失败但接口状态码是 200？

GGPT 的购买接口业务失败时也可能返回 HTTP 200，需要看接口 JSON 里的 `ret` 和 `msg`。

例如：

```json
{"ret": -1, "msg": "User bonus not enough."}
```

表示账号 G 值/魔力不足。

### 保存配置会直接购买吗？

不会直接无脑购买。它会先访问勋章页面检查当前是否已经购买。页面显示已购买时，只更新预计下次购买时间；只有未购买或已到期时才提交购买接口。

### 为什么预计下次购买不是 7 天后？

如果本次没有购买成功，也没有从页面解析到当前勋章到期时间，插件无法知道真实的 7 天周期，会先显示下一次每日刷新时间。

### 为什么没有自动购买？

请检查：

- 插件是否已启用
- GGPT 站点 Cookie 是否有效
- 勋章 ID 是否正确
- 账号 G 值/魔力是否足够
- MoviePilot 是否重启过，重启后建议重新打开插件确认状态

### 勋章 ID 变了怎么办？

在插件配置页修改“勋章 ID”并保存配置。当前“疯狂星期四”勋章 ID 为 `35`。
